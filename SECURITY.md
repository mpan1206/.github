# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in any of my projects, please report it by [opening a GitHub Issue](https://github.com/mpan1206/.github/issues/new) with the **\[Security\]** prefix in the title.

> **Note:** For highly sensitive vulnerabilities, you may also use [GitHub's private vulnerability reporting](https://docs.github.com/en/code-security/security-advisories/guidance-on-reporting-and-writing-information-about-vulnerabilities/privately-reporting-a-security-vulnerability) if available on the specific repository.

## What to Include

When reporting a vulnerability, please include:

- A description of the vulnerability
- Steps to reproduce the issue
- The potential impact
- Any suggested fixes (if applicable)

## Response

- I will acknowledge receipt of your report within **48 hours**.
- I will provide an initial assessment within **7 days**.
- I will work on a fix and coordinate disclosure as appropriate.

## Supported Versions

Unless otherwise stated in a specific project, only the **latest release** is actively maintained and receives security updates.
